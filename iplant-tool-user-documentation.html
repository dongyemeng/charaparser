CharaParser for iPlant

CharaParser for iPlant consists of two tools, CharaParser Learn and CharaParser Markup.
CharaParser Learn takes morphological descriptions of the biodiversity domain as input and extracts identified structure, character and other terms of possible interest. The identified terms are sent to the OTO Lite for user review and further specification. The output of CharaParser Learn is a text file containing a direct link to OTO Lite, where the respective terms can be reviewed.
CharaParser Markup takes the same morphological descriptions of the biodiversity domain as input. Using an established domain glossary, in combination with the user reviewed terms extracted by CharaParser Learn, the morphological descriptions are semantically marked up. The output for CharaParser Markup are XML files containing the semantically markedup descriptions.

CharaParser Learn Input

  • ID
  • Configuration
  • XML Input files according to the defined schema

CharaParser Learn Output

  • TermReview.txt: File containing direct link to OTO Lite to review newly discovered terms
  • config.txt: The configuration used
  • error.log: Records occured errors while parsing

CharaParser Markup Input

  • ID
  • Configuration
  • XML Input files according to the defined schema

CharaParser Markup Output

  • *.xml: The marked up result files. These will be according to the defined schema.
  • config.txt: The configuration used
  • error.log: Records occured errors while parsing
