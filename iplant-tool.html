CharaParser for iPlant

CharaParser for iPlant consists of two tools, CharaParser Learn and CharaParser Markup.
CharaParser Learn takes morphological descriptions of the biodiversity domain as input and extracts identified structure, character and other terms of possible interest. The identified terms are sent to OTO Lite for user review and further specification. The output of CharaParser Learn is a text file containing a direct link to OTO Lite, where the respective terms can be reviewed.
CharaParser Markup takes the same morphological descriptions of the biodiversity domain as input. Using an established domain glossary, in combination with the user reviewed terms extracted by CharaParser Learn, the morphological descriptions are semantically marked up. The output for CharaParser Markup are XML files containing the semantically marked up descriptions.

Installation and Dependencies

CharaParser Learn and Markup can be downloaded here and here. Both come in form of runnable Java .jar files (learn.jar, markup.jar) compiled for Java 1.7, packed with additionally needed files. All delievered files are to be placed in the same directory and structure as they are delivered.

CharaParser depends on the following software:

  • Java 1.7
  • MySQL 5.5
  • WordNet 3.0
  • Perl (tested with Perl 5.16) and the modules
    • DBI
    • DBD::MySQL

To execute CharaParser Learn and CharaParser Markup in the iPlant Discovery Environment, they are delivered with two Bash scripts (learn.sh, markup.sh). The scripts take care of moving log files into a directory accessible to the CharaParser developers after execution.

Command-Line Options

CharaParser Learn

-c <arg> * Configuration to use. Will depend on the type of dataset to process. Possible configurations: Plant, Hymenoptera, Algea, Porifera, Fossil.
-n <arg> * MySQL host
-p <arg> * MySQL port
-d <arg> * Database name to use
-u <arg> * Database user to use
-s <arg> * Database password to use
-z <arg> * Database table prefix to use. This has to be an ID that has not been used by another invocation of CharaParser Learn before. The ID has to be recalled for the corresponding invocation of CharaParser Markup.
-i <arg> * Directory containing the input files. Each file has to be valid according to the defined schema.
-h,--help Shows the help
* marks required parameters

CharaParser Markup

-c <arg> * Configuration to use. Will depend on the type of dataset to process. Possible configurations: Plant, Hymenoptera, Algea, Porifera, Treatise.
-n <arg> * MySQL host
-p <arg> * MySQL port
-d <arg> * Database name to use
-u <arg> * Database user to use
-s <arg> * Database password to use
-z <arg> * Database table prefix to use. This has to be the corresponding ID that has been used for the invocation of CharaParser Learn with the same input dataset.
-i <arg> * Directory containing the input files. Each file has to be valid according to the defined schema.
-t <arg> Use multi-threading to compute the result. <arg> is to be given in the form of <max. number of concurrent sentence chunkers,max. number of concurrent description extractors>. Note that no space is allowed between the arguments and their separating comma, e.g. -t 10,2
-h,--help Shows the help
* marks required parameters
