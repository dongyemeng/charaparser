CharaParser for iPlant

CharaParser for iPlant consists of two tools, CharaParser Learn and Charaparser Markup. Both come in form of runnable Java .jar files compiled for Java 1.7.

Dependencies

Charaparser Learn and Charaparser Markup have the following software dependencies:
  • MySQL
  • Perl (tested with Perl 5.16) and the modules
    • DBI
    • DBD::MySQL

Command-Line Options

Charaparser Learn:
-c,--config <arg> Configuration to use. Will depend on the type of dataset to process. Possible configurations: Plant, Hymenoptera, Algea, Porifera, Treatise.
-dbh,--database-host <arg> MySQL host
-dbu,--database-user <arg> <arg> Database user to use
-dbp,--database-password <arg> Database password to use
-dbtp,--database-table-prefix <arg> * Database table prefix to use. This has to be an ID that has not been used by another invocation of Charaparser Learn before. The ID has to be recalled for the corresponding invocation of CharaParser Markup.
-i,--input <arg> * Input file or directory
-style,--style mapping <arg> Optional style mapping to use for Word file input
-h,--help Shows the help
* marks required parameters Charaparser Markup:
-c,--config <arg> Configuration to use. Will depend on the type of dataset to process. Possible configurations: Plant, Hymenoptera, Algea, Porifera, Treatise.
-dbh,--database-host <arg> MySQL host
-dbu,--database-user <arg> <arg> Database user to use
-dbp,--database-password <arg> Database password to use
-dbtp,--database-table-prefix <arg> * Database table prefix to use. This has to be the corresponding ID that has been used for the invocation of CharaParser Learn with the same input dataset.
-i,--input <arg> * Input file or directory
-style,--style mapping <arg> Optional style mapping to use for Word file input
-p,--multi-threading <arg> Use multi-threading to compute the result. Give <arg> in the form of <number of threads per sentence chunking, number of threads per description extractor>
-h,--help Shows the help
* marks required parameters
