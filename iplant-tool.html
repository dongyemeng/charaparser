CharaParser for iPlant

CharaParser for iPlant consists of two tools, CharaParser Learn and Charaparser Markup.
CharaParser Learn takes morphological descriptions of the biodiversity domain as input and extracts identified structure, character and other terms of possible interest. The identified terms are sent to the Ontology Organizer (OTO) for user review and further specification. The output of CharaParser Learn is a text file containing a direct link to OTO, where the respective terms can be reviewed.
Charaparser Markup takes morphological descriptions of the biodiversity domain as input. Using an established domain glossary, in combination with the user reviewed terms extracted by Charaparser Learn, the morphological descriptions are semantically marked up. The output for Charaparser Markup are XML files containing the semantically markedup descriptions.

Installation and Dependencies

CharaParser Learn and Markup come in form of runnable Java .jar files compiled for Java 1.7, packed with additional files. They depend on the following software:
  • Java 1.7
  • MySQL
  • Perl (tested with Perl 5.16) and the modules
    • DBI
    • DBD::MySQL

Command-Line Options

CharaParser Learn

-c,--config <arg> Configuration to use. Will depend on the type of dataset to process. Possible configurations: Plant, Hymenoptera, Algea, Porifera, Treatise.
-dbh,--database-host <arg> MySQL host
-dbu,--database-user <arg> <arg> Database user to use
-dbp,--database-password <arg> Database password to use
-dbtp,--database-table-prefix <arg> * Database table prefix to use. This has to be an ID that has not been used by another invocation of CharaParser Learn before. The ID has to be recalled for the corresponding invocation of CharaParser Markup.
-i,--input <arg> * Input file or directory
-style,--style mapping <arg> Optional style mapping to use for Microsoft Word file input
-h,--help Shows the help
* marks required parameters

CharaParser Markup

-c,--config <arg> Configuration to use. Will depend on the type of dataset to process. Possible configurations: Plant, Hymenoptera, Algea, Porifera, Treatise.
-dbh,--database-host <arg> MySQL host
-dbu,--database-user <arg> <arg> Database user to use
-dbp,--database-password <arg> Database password to use
-dbtp,--database-table-prefix <arg> * Database table prefix to use. This has to be the corresponding ID that has been used for the invocation of CharaParser Learn with the same input dataset.
-i,--input <arg> * Input file or directory
-style,--style mapping <arg> Optional style mapping to use for Word file input
-p,--multi-threading <arg> Use multi-threading to compute the result. <arg> is to be given in the form of <max. number of concurrent sentence chunkers, max. number of concurrent description extractors>
-h,--help Shows the help
* marks required parameters
